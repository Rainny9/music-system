一、项目功能与总体设计
1.1 核心功能
音乐播放
在线播放：前端使用 audio 播放后端提供的音频 URL。
歌曲资源管理
后台：增删改查歌曲（名称、歌手、专辑、分类/风格、封面、音频文件）。
歌曲评论、收藏
评论：登录用户对歌曲发表评论、查看评论列表。
收藏：登录用户收藏/取消收藏歌曲，查看自己的收藏列表。
上传和下载
上传：用户或管理员上传音频文件。
下载：点击下载按钮，后端返回音频文件。
分类与检索
按 歌名 / 歌手 / 分类 / 标签 进行搜索和筛选。
前台页面
首页歌曲列表、分类筛选、搜索、播放页、评论区、收藏按钮、用户登录注册。
后台页面
管理员登录、歌曲管理、歌单（可选）、用户管理（简单化，可选）。
二、技术选型与系统架构
2.1 技术选型（适合初学者）
后端（Python）
框架：Flask（轻量、简单，教程多）
ORM：SQLAlchemy（操作数据库更方便）
数据库：初期可用 SQLite（无需单独安装服务），后期可以换 MySQL
其他：
flask-cors：处理前端跨域
Werkzeug：文件上传处理（Flask 内置）
前端（Vue）
Node.js + npm
构建工具：Vite + Vue 3（官方推荐）
路由：vue-router
HTTP 请求：axios
UI：先用原生 HTML+CSS+简单样式，后期你可以换成 Element Plus 等组件库
2.2 系统架构（前后端分离）
前端（Vue 应用）
运行在：http://localhost:5173（Vite 默认）
功能：页面渲染、调用后端 API、音频播放
后端（Flask API）
运行在：http://localhost:5000
功能：登录注册、歌曲CURD、评论、收藏、文件上传下载、分类搜索
静态文件存储
音频文件和封面图片存本地目录，如 uploads/songs/、uploads/covers/
架构简化理解：
浏览器中的 Vue 网页 ←(HTTP JSON 请求)→ Flask 后端 ←→ 数据库 + 本地文件
